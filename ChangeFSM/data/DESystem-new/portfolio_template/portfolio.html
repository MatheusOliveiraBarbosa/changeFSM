Portfolio Overview

  • Design
    • Architecture Diagram - The architecture diagram describes the objects present in the system, the replication of distributed objects, and the communication channels between objects.
    • Use Cases - The use case diagram describes the ways that agents in the system can interact with the elevator and shows relationships between these methods of interaction.
    • Scenarios and Sequence Diagrams - The scenarios demonstrate a possible set of actions for each use case and the sequence diagrams demonstrate the sequence of actions in a UML diagram.
    • Requirements I - System Object Descriptions and Message Dictionary - A list of the sensors and actuators in the system and a message dictionary describing the messages that are required for the various controllers.
    • Requirements II - Distributed Controller Requirements - Contains detailed behaviors and requirements for all the of the controllers.
  • Traceability
    • Sequence Diagrams to Requirements Traceability - A traceability chart mapping each of the sequence diagram arcs to at least one requirement and vice versa.
    • Requirements to Contstraints Traceability - A traceability chart mapping the requirements to the constraints.
    • Statecharts to Code Traceability - This document shows where each transition described in the controller statecharts is implemented in the code.
  • Implementation
    • Elevator Control Package - Contains all the java implementations of the different controllers in the elevator.
  • Test
    • Unit Test Log - Unit tests that injects messages to test all the states and transitions of the controller.
    • Unit Test Summary File - parseable list of unit test files
    • Integration Test Log - Tests that injects messages to test states and transitions for at least one controller in a sequence diagram.
    • Integration Test Summary File - parseable list of integration test files
    • Acceptance Test Log - Tests that injects passengers to test working of overall elevator system.
    • Fault Tolerance Test Log - Not Implemented
  • Log Files
    • Issue Log - This log contains any issues that were found during the course of the project and the log is updated once issues are fixed.
    • Improvements Log - This log contains suggestions for improving the project in later phases.
    • Peer Review Log - This log contains a record of all the peer reviews done in the course of this project.
  • Scheduling
    • Network Schedule -  Schedule messages that are sent onto the network such they they are limited to a particular bandwidth.
  • Fault Tolerance
    • FMEA - Not Implemented
    • FTA - Not Implemented
  • Verification
    • High Level Requirements Verification Framework - Uses a runtime monitor to monitor whether the high level requirements are being satisfied by the elevator.
  • Presentations
    • Midterm Presentation Slides - Slides for the midterm presentation, covering up to Project 7 and the DoorControl Controller.
    • Final Presentation Slides  - Slides for the final presentation, covering final design of elevator, focuse on DoorControl Controller.
